# Zotero AI Butler

## 它是什么

Zotero AI Butler（Zotero AI 管家）是一个 Zotero 插件，调用大模型自动精读论文库里的论文，并总结为条理清晰的 Markdown 笔记保存到 Zotero 条目下。

它的核心思路是：

> 你把论文丢进 Zotero，管家自动帮你读完、总结成笔记，让你"十分钟完全了解"这篇论文。

仓库地址：[https://github.com/steven-jianhao-li/zotero-AI-Butler](https://github.com/steven-jianhao-li/zotero-AI-Butler)

## 适合谁

- 文献太多读不过来
- 读完后很快忘记，需要回顾
- 论文太长，即使有翻译也难以抓住重点
- 想用 AI 批量处理积压的旧论文

## 核心功能

- 自动化处理：自动扫描新添加的 PDF 文献，生成 Markdown 笔记
- 多模态支持：支持 Base64 上传 PDF，数学公式、图表和复杂排版无损提取，中英文论文全支持
- 一图总结：用 Nano Banana Pro 为每篇论文生成学术海报式图片
- 思维导图：自动生成论文思维导图，支持放大缩小、导出 PNG 和 OPML 格式
- 多文献综述：右键分类即可对多篇论文进行综合分析，生成文献综述报告
- 沉浸阅读：内置 AI 管家侧边栏，支持 LaTeX 公式渲染和大模型追问
- 多轮对话：支持单次对话、多轮拼接、多轮总结三种精读模式
- 侧边栏追问：完整追问（保存记录）和快速提问（不保存记录）两种模式
- 开源平台：所有模型提示词可自定义，支持多个大模型 API

## 安装

1. 访问 [GitHub Releases 页面](https://github.com/steven-jianhao-li/zotero-AI-Butler/releases)
2. 下载最新的 `.xpi` 文件
3. 打开 Zotero 桌面端，点击顶部菜单：`工具` → `插件`
4. 将下载好的 `.xpi` 文件拖拽到插件窗口中，完成安装

## 快速配置

1. 在任意论文上右键 → `AI管家仪表盘` → 打开 `设置` 选项卡
2. 配置 API：填入你的 API Key（推荐 Gemini），点击 `测试连接` 确保网络通畅
3. 开启自动扫描：转到 `界面设置` 选项卡，勾选 `自动扫描新文献`

## 三种任务触发方式

### 方式一：右键唤醒（即时处理）

在论文条目上右键，选择 `召唤AI管家进行分析`，任务将立即进入队列。

多轮对话重新精读：右键菜单中选择 `AI管家多轮对话重新精读`，可选 `多轮拼接` 或 `多轮总结`。

### 方式二：自动巡航（新文献自动处理）

在 `仪表盘` → `界面设置` 中开启 `自动扫描新文献`。开启后，拖入新论文时管家会自动开始分析。设置会自动保存。

### 方式三：批量处理（旧文献回溯补充）

在 `仪表盘` 中点击 `扫描未分析论文`，AI 管家会自动找出所有没有笔记的论文，按 Zotero 目录结构排列。勾选需要补充的论文，点击 `添加到队列` 即可后台处理。

## 与其他工具的配合

Zotero AI Butler 和本指南中其他工具的关系：

- Trellis：用于管流程和代码上下文（如果你要改插件源码）
- Claude Code / Codex / Gemini：用于理解和修改插件源码
- cc-switch：用于切换调用大模型时的 provider 配置
- CPA / CLIProxyAPI：可作为 AI 管家的 API 代理层，统一管理多平台 API Key

如果你只使用插件而不改源码，只需配置好 API Key 即可，不需要 Trellis 或其他 CLI 工具。

## 常见坑

- API Key 没配置或配置错误
- 忘记在设置中开启 `自动扫描新文献`，导致拖入论文后没有自动分析
- 多模态处理（Base64）模式下 API 费用比纯文本高
- 批量处理时 API 调用速度设置过高导致限流
- Ollama 模式下不支持多模态，需使用文本提取或 MinerU 处理 PDF

## 推荐配置方案

新手推荐：

- 模型：Google Gemini 3 Pro（多模态能力强，总结易懂）
- 模式：单次对话（Token 消耗最少，适合快速阅读）
- 开启自动扫描，一劳永逸

进阶推荐：

- 用 Gemini 做快速通读
- 用多轮总结模式做深度研读
- 用一图总结快速建立记忆点
