# MCP

## 它是什么

MCP 是 Model Context Protocol 的缩写。你可以把它理解成一种标准协议，用来让 AI 客户端连接到外部工具、数据源或服务。

对新手来说，最重要的一点是：

- **MCP** 是“协议”
- **MCP Router** 是“管理 MCP 服务器的应用”

它们不是一回事。

## 它解决什么问题

如果没有统一协议，不同 AI 工具接外部能力时，往往要写各自的连接方式。MCP 的目标就是让这种连接方式更统一。

你可以把它类比成：

- 客户端：Claude、Cursor、Windsurf 等 AI 工具
- 服务器：提供具体能力的 MCP server
- 协议：让两边说同一种“语言”

## 你会在什么地方遇到它

你会在这些地方看到 MCP：

- 给 AI 客户端接数据库、文件系统或内部工具
- 给模型加自定义工具
- 在 cc-switch 或其他管理工具里管理 MCP servers
- 在桌面管理器里启用、禁用或分组 MCP 配置

## 新手先记住什么

你不需要一开始就理解全部细节，只要记住：

1. MCP 是连接标准
2. MCP server 是能力提供方
3. AI 客户端是使用这些能力的一方
4. MCP Router 是帮你管理这些 server 的工具

## 常见坑

- 把 MCP 当成具体应用，而不是协议
- 把 MCP Router 和 MCP server 混为一谈
- 以为所有 AI 工具都自动支持同样的 MCP 配置

## 一句话解释

> MCP 是让 AI 工具和外部能力对接的统一协议。
